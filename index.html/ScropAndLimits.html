Scope and Limits

• Tools and Technologies

For our healthy lifestyle application, we decided to go with a hybrid application development cycle as it has a lower barrier of entry compared to native application development. Since it will be available on both iOS and the Google Play store, our application would be exposed to a larger market.

Application development

We will be using the latest React Native (version 0.63), a development framework that uses Javascript as its main coding language to develop our application for iOS and Android devices. React Native is open source so it is free of charge to use, it also supports both hybrid and native application development. The integrated development environment (IDE) we will be using to develop our application will be IntelliJ Idea (Version: 2020.3.1). Students can sign up with their email address for a free license of IntelliJ Idea with their registered school email address.

UI Mockup

For our UI design process, we would use Figma UI design tool to draft up a user interface for the app to test our product. We chose to go with Figma as its preview functions allow the users to navigate the mock-up app as if they are using a real application.

For our basic uses of consumer interaction with the device, the Figma free license is sufficient. If the need arises, we can sign up for the ‘Professional’ tier for $12 a month so that our members can collaborate in the UI design in real-time.

• Testing
Methodology

Our mobile application process development will be very similar to the ‘agile method’ of software development where test users are called in to try out the application whenever we develop a new build or add a function. This allows us to tailor the application to suit the user’s needs as we are able to get the user’s feedback in a timely manner instead of waiting till the application is fully developed to carry out user testing.

Users will be divided into the 4 active lifestyle categories; sedentary (little to no exercise), light, moderate and vigorous physical activity. Firstly, we would take their original weight at the start of the program and we would then ask them to track their weights throughout the week for a period of 3 months. The results would then be tallied up and the average weight loss for each fitness group would then be forwarded to our data analysts to find out if our proposed program is effective and how it can be improved.

Testing Periods

Keeping in mind that since our mobile application is a healthy lifestyle application, we would need to monitor the user’s fitness progression over an extended period of time (at least 3 months). It is due to the fact that a human body does not change its size overnight and a healthy and consistent diet and exercise regime is important if the user wants to achieve their dream body.

User testing sample size

Usability testing websites suggest using the 5-person rule for usability testing but we disagree on such a small sample size. We believe that a 40-man user sample size, 10 users for each of the lifestyle groups, would be sufficient. Since we have 10 people per lifestyle group, we would then have a reliable average when measuring their weights at the end of the diet period.

First and foremost, the main aim of our application is for users to reach their desired body form and weight. We would certainly need a certified dietician/fitness expert who has a good understanding of human physiology and metabolism to help us to design programs that tailors to the user’s goal.

Secondly, since we are funded, we would request for a legal legislator to go over what regulations we are supposed to follow when developing and testing our application before we launch it to the market. Legal actions can be costly in the long run so we think that it is a wise choice to invest in a legislator as it is better to be safe than sorry. It is preferable if he/she are well versed in mobile application development as they would be able to understand technical jargons better when meeting with the development team.

Thirdly, we would like to have a data analyst on the team so that the team is always informed on how our program is affecting the user’s body goal result. Since our coding team would be too busy trying to develop a stable build for the application, having a data analyst to produce graphs for the dietician/fitness staff helps them to visualise the next steps to improve the fitness program. We can also know what the user likes from the application and its pain point from surveying them once in a while throughout their program.

Finally, an experienced full stack mobile developer, one that programs both the front-end (user experience) and the back-end (running the server, developing the application) would be very valuable to the team. He/She would be responsible for developing the application and incorporating the feedback from the three positions mentioned above.
